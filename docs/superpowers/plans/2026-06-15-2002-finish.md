## Task 3: Full Korean names via Wikidata (script; run deferred)

**Files:** Create `scripts/fetch-konames.ts`

For each unique player in `data/raw/2002/player_appearances.csv` lacking a curated name in `data/mappings/players.ko.json`: resolve to a Wikidata QID (search by `${given} ${family}` via `https://www.wikidata.org/w/api.php?action=wbsearchentities&search=...&language=en&type=item&format=json`), then fetch the entity's `labels.ko.value` via `wbgetentities&props=labels&languages=ko`. Throttle (≤5 req/s), compliant User-Agent `WorldCupArchive/1.0 (+https://worldcup.minseok91.cloud)`. Merge found ko labels into `players.ko.json` `names`. Cache resolved QIDs in `data/raw/2002/wikidata-cache.json` to make re-runs cheap. Then re-run `npm run data:generate && npm run data:validate`.
Run as a deferred throttled batch (≈700 players); not inline.

## Task 4: Commons photos (script; run deferred)

**Files:** Create `scripts/fetch-photos.ts`, `data/generated/2002/images.json`

For each player QID (from Task 3 cache): Wikidata `P18` → Commons filename → `https://commons.wikimedia.org/w/api.php?action=query&prop=imageinfo&titles=File:<f>&iiprop=url|extmetadata&iiurlwidth=320&format=json`. Store `{ playerId: { url, descriptionurl, artist, licenseShortName, licenseUrl } }` in `images.json`; mirror the 320px thumb to `public/players/<playerId>.jpg`. Throttle + User-Agent as Task 3. Then update `FallbackAvatar`'s parent (a new `PlayerAvatar` wrapper) to show the photo if present, else the existing fallback; show attribution on player pages. Coverage expectation: ~68% (2002) — fallback covers the rest. Deferred batch.
